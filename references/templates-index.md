# Templates Index

This skill has a built-in template library, organized into directories by methodology/purpose. Each template is Markdown with placeholders, rendered with
`scripts/render.py --template <template> --data <data.yaml> --out <artifact.md>`.

> **Field contract**: each template below lists the data keys required for rendering (top-level YAML keys). When a sub-Agent / user fills in data, they
> must provide these keys (if missing, they render empty, but the consistency check may still fail, so fill unknown items with "(TBD)").

> **Chart rendering**: all chart-type deliverables (WBS Gantt, schedule Gantt, burndown, dependency map, governance map, macro-micro alignment map)
> are rendered using Mermaid code blocks, directly visualizable in Mermaid-supporting viewers such as GitHub / VS Code / Typora.

## _macros.md
Shared fragments (table headers, version badge, status legend). Templates can call them with `{{#each}}`; to reuse a fragment,
the master splices in the `_macros.md` content before rendering, or provides equivalent fields in the data.

## Extension Method
Adding a methodology/artifact = add a `.md` template in the corresponding directory (following render.py syntax) + register its data keys in this table,
with no need to modify the engine or SKILL.md.

## Standard Initiation / Governance Suite (recommended default output)
- **Any project initiation**: project_charter + stakeholder_register + raci + communication_plan
  + plan template (wbs / backlog / iteration_plan) + risk_register + raid_log.
- **waterfall / hybrid**: wbs (estimate + dependency network + DoD) + stage_gate_review + quality_plan +
  schedule_health analysis; execution/monitoring phases require status_report + evm analysis.
- **agile / iteration**: product_backlog / iteration_plan + sprint_plan + definition_of_done +
  burndown + retro.
- **program / hybrid (change control required)**: program_charter + portfolio_dashboard + dependency_map +
  benefits_realization + **change_log** (CCB); hybrid additionally requires at least 1 micro-layer plan (sprint/backlog/iteration).
- **Operational control (operational, mandatory)**: planning→review→**baseline**→control gate→operational is a mandatory serial state machine;
  waterfall / hybrid **must** `baseline.py --freeze` before entering execution/monitoring (producing baseline_record + control_register),
  after which `control_engine.py` runs periodically on the `control.cadence` cycle to generate the control_report. See references/lifecycle.md §5.
- When items in the above suites are missing, `consistency_check.py` will **block** or **warn** per control-level rules (see the script documentation).
